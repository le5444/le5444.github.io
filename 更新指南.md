# 织梦写作台 · 更新部署指南

> 想给手机上的 App 更新最新版?按下面步骤来。

---

## 流程总览

```
改代码 → 重新打包 → 上传 GitHub 网页覆盖 → 等 1-3 分钟 → 手机刷新
```

---

## 第 1 步:在本地改代码

随便用 VS Code / Cursor 之类的编辑器打开:

```
C:\Users\30865\Desktop\织梦写作台-项目打包-20260517-145943
```

在 `src/` 里改你想改的东西。

---

## 第 2 步:重新打包

打开终端(PowerShell / cmd),`cd` 到项目目录,然后跑:

```bash
npm run build:pwa
```

等几秒钟看到 `✓ built in X.XXs` 就好了。

这一步会**重新生成** `dist-pwa/` 文件夹里的所有文件。

> ⚠️ **如果跳过这一步直接上传 `src/`**,GitHub 不会帮你编译,网站会坏。**必须先 build**。

---

## 第 3 步:打开 GitHub 仓库

浏览器打开:

👉 **https://github.com/le5444/le5444.github.io**

(如果没登录,先登录。)

---

## 第 4 步:删掉旧的 assets 文件夹(可选但推荐)

Vite 每次打包后 `assets/` 里的 JS/CSS 文件名带不同哈希(比如 `index-Bo8JmCBf.js` → `index-X9aLpQrs.js`),所以**旧的不会被自动覆盖,会越攒越多**。

- 在仓库主页点进 **`assets`** 文件夹
- 右上角点 **⋮(三个点)** 或文件夹设置图标 → 选 **「Delete directory」/「删除目录」**
- 滑到底 → **「Commit changes」**

如果你不在乎仓库变大,这一步可以跳过,**功能不受影响**。

---

## 第 5 步:上传新文件

1. 仓库主页点 **「Add file ▾」** → **「Upload files」**
2. 打开你电脑上的:
   ```
   C:\Users\30865\Desktop\织梦写作台-项目打包-20260517-145943\dist-pwa\
   ```
3. **全选里面所有内容**(Ctrl+A):9 个文件 + `assets` 文件夹
4. **整批拖**到网页虚线框里
5. 等上传条走完
6. 滑到底,**「Commit changes」**区域:
   - 标题写一句话比如「更新:加了 XX 功能」
   - 点绿色 **「Commit changes」** 按钮

---

## 第 6 步:等部署 & 手机刷新

1. 等 **1-3 分钟**(可以去仓库 **Actions** Tab 看进度,出绿色 ✅ 就好了)
2. 电脑浏览器先打开 https://le5444.github.io/ 验证:看到改动 = 成功
3. **手机看不到新版**?三种办法:
   - **等**:Service Worker 自动检测更新,通常下次或下下次打开就是新版
   - **快办法**:长按 App 图标 → 「应用信息 / App 信息」→ 「清除缓存」→ 重新打开
   - **保险**:删掉桌面图标 → Safari/Chrome 重新访问 → 重新「添加到主屏幕」

## 常见问题

**Q: 我能不能用 Cursor / VS Code 的"上传到 GitHub"按钮?**
A: 可以,但要先 `npm run build:pwa` 生成新的 `dist-pwa`,然后用 git 推送。比网页拖快,但要懂 git。

**Q: 网址打开是白屏 / 一直转圈?**
A: 90% 是因为 `assets/` 里新旧文件名对不上,或者 `index.html` 没更新。把仓库里 `assets/` 删干净,重新上传 `dist-pwa/` 里所有内容。

**Q: 改了 manifest 或图标,手机图标没换?**
A: 必须卸载 App 重新「添加到主屏幕」,手机系统不会主动重新读 manifest。

**Q: 我可以不更新吗?**
A: 完全可以。GitHub Pages 上的版本会一直在,不更新就不变。
